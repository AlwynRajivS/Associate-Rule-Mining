📖 Fundamentals
Association Rule Mining
Association Rule Mining (ARM) discovers interesting relations between variables in large datasets. It identifies frequent if-then patterns and quantifies their strength using support, confidence, and lift.
Core Concepts
📦
Itemset
A collection of one or more items. e.g. {HighMarks, HighAttendance}.
📏
Frequent Itemset
Itemset whose support ≥ minimum support threshold. Only these generate valid rules.
➡️
Association Rule
An implication X → Y where X (antecedent) and Y (consequent) are disjoint.
🔍
Strong Rule
A rule satisfying both minimum support AND minimum confidence thresholds.
Key Metrics
Support(X→Y) = count(X∪Y) / N
Confidence(X→Y) = count(X∪Y) / count(X)
Lift(X→Y) = Confidence / Support(Y)
Conviction = (1 - Supp(Y)) / (1 - Conf)
Lift > 1
Positive correlation — appear together more than by chance.
Lift = 1
Independent — no predictive power.
Lift < 1
Negative correlation — appear less than chance.
Types of Rules
Single-Dimensional
  • Rules involve one attribute
  • buys(bread) → buys(butter)
  • Simple, fast to mine
Multi-Dimensional
  • Rules span multiple attributes
  • age=young ∧ dept=CS → marks=high
  • Richer, more actionable patterns
⚙️ Algorithm
The Apriori Algorithm
Apriori (1994, Agrawal & Srikant) mines frequent itemsets using the anti-monotonicity principle: any subset of a frequent itemset must also be frequent — enabling massive pruning of the search space.
Step-by-Step Process
1
Find Frequent 1-Itemsets (L₁)
Scan database, count each item. Remove items below min_sup → L₁.
min_sup=40% | Items: A:8✓ B:6✓ C:3✗ → L₁={A,B}
2
Generate Candidates (Cₖ)
Join Lₖ₋₁ with itself to produce all k-sized pairs.
C₂ = {A,B}, {A,C}, {B,C} → scan → count
3
Prune (Apriori Property)
If any subset is infrequent → prune the whole superset.
{A,B}:5✓ {A,C}:2✗ → L₂={A,B}
4
Repeat Until No New Itemsets
Continue generating Cₖ, filtering to Lₖ until empty.
5
Generate Rules
From each frequent itemset I (|I|≥2), generate all X→Y rules. Keep if conf ≥ min_conf.
From {A,B}: A→B conf=5/8=62.5%✓ | B→A conf=5/6=83%✓
Limitations vs Alternatives
Apriori Weaknesses
  • Multiple database scans
  • Many candidate itemsets
  • Memory-intensive on dense data
Better Alternatives
  • FP-Growth: tree-based, 2 scans only
  • ECLAT: vertical data format
  • Max-Miner: finds maximal itemsets
🌳 Concept
Multilevel Association Mining
Mining at various levels of a concept hierarchy. Higher levels yield general rules with stronger support; lower levels yield specific rules with richer context.
Concept Hierarchy Example
Education Dataset
Level 0: All Students
Level 1 (Dept): CS, ECE, MECH
Level 2 (Gender): Male, Female
Level 3 (Residence): Hostel, Day Scholar

L1 rule: CS → HighMarks (sup=45%)
L2 rule: CS ∧ Female → HighMarks (sup=28%)
L3 rule: CS ∧ Female ∧ Hostel → HighMarks (sup=14%)
Support Strategy
Uniform Support
  • Same threshold all levels
  • Simple, may miss rare rules
Reduced Support
  • Lower threshold at deeper levels
  • Captures specific patterns
🔷 Concept
Multidimensional Association Mining
Rules spanning two or more attributes simultaneously — revealing cross-attribute patterns invisible in single-column analysis.
Rule Types
Inter-Dimension
Each attribute appears once: age=young ∧ dept=CS → marks=high
Hybrid Rules
Some attributes appear multiple times at different levels.
Heatmap Interpretation
Reading the Cross-Tabulation Matrix
Cell(row,col) = count where both attributes co-occur.
Darker cells = more records = stronger co-occurrence.
Hot spots reveal high co-occurrence pairs — ideal rule candidates.
📐 Reference
Complete Metrics Reference
All metrics used in association rule mining with formulas, ranges, and interpretation.
Formulas & Ranges
Support(X→Y) = |X∪Y|/N [0,1] Higher = more frequent
Confidence(X→Y) = |X∪Y|/|X| [0,1] Higher = more reliable
Lift(X→Y) = Conf/Supp(Y) [0,∞) >1 = positive corr
Leverage(X→Y) = Supp(X∪Y)-Supp(X)·Supp(Y) [-1,1]
Conviction = (1-Supp(Y))/(1-Conf) [0,∞)
Choosing Thresholds
High Sup (>50%)
Very common patterns, few rules, high quality.
Medium Sup (20-50%)
Balanced discovery. Best for most datasets.
Low Sup (<20%)
Rare pattern discovery. Use with high confidence.
High Conf (>80%)
Only very reliable rules. Best for predictions.
