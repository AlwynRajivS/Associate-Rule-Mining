📖 Fundamentals
Association Rule Mining
Association Rule Mining (ARM) is a rule-based machine learning method for discovering interesting relations between variables in large datasets. It identifies frequent if-then patterns and quantifies their strength using statistical measures like support and confidence.
Core Concept
📦
Itemset
A collection of one or more items. e.g. {HighMarks, HighAttendance} is a 2-itemset.
📏
Frequent Itemset
An itemset whose support ≥ minimum support threshold. Only these generate valid rules.
➡️
Association Rule
An implication X → Y where X (antecedent) and Y (consequent) are disjoint itemsets.
🔍
Strong Rule
A rule that satisfies both minimum support AND minimum confidence thresholds simultaneously.
Key Metrics
Support(X → Y) = P(X ∪ Y) = count(X and Y) / total_transactions
Confidence(X → Y) = P(Y | X) = count(X and Y) / count(X)
Lift(X → Y) = Confidence(X → Y) / Support(Y)
Conviction(X → Y) = (1 - Support(Y)) / (1 - Confidence(X → Y))
Lift > 1
X and Y appear together more than by chance. Positive correlation.
Lift = 1
X and Y are independent. The rule has no predictive power.
Lift < 1
X and Y appear together less than chance. Negative correlation.

Types of Association Rules
Single-Dimensional
  • Rules involve one attribute
  • e.g. buys(bread) → buys(butter)
  • Simple, fast to mine
  • Less contextual insight
Multi-Dimensional
  • Rules span multiple attributes
  • e.g. age=young ∧ income=high → buys(laptop)
  • Richer, more actionable patterns
  • Used in this platform
⚙️ Algorithm
The Apriori Algorithm
Apriori (1994, Agrawal & Srikant) is the classic frequent itemset mining algorithm. Its key insight: any subset of a frequent itemset must also be frequent — the "Apriori property" allows massive pruning of the search space.
The Apriori Property (Anti-Monotonicity)
🔑 Key Insight
If itemset {A, B} is infrequent, then ALL supersets like {A, B, C}, {A, B, D} must also be infrequent.

This means we can prune entire branches of the search tree without checking them — drastically reducing computation.
Step-by-Step Process
1
Scan & Find Frequent 1-Itemsets (L₁)
Count occurrences of every single item. Remove items below minimum support threshold (min_sup). The surviving items form L₁.
min_sup = 0.40 (40%)
Count: {HighStudy:8, CS:6, Hostel:7, HighMarks:7, LowAttend:3}
Prune: LowAttend (3/10=30% < 40%) ✗
L₁ = {HighStudy, CS, Hostel, HighMarks}
2
Generate Candidate 2-Itemsets (C₂)
Join L₁ with itself to produce all pairs. These are candidates — not yet verified as frequent.
C₂ = {HighStudy,CS}, {HighStudy,Hostel}, {HighStudy,HighMarks},
{CS,Hostel}, {CS,HighMarks}, {Hostel,HighMarks}
3
Scan Database → Find Frequent 2-Itemsets (L₂)
Count each candidate pair in actual data. Remove pairs below min_sup.
Counts: {HighStudy,HighMarks}:7✓ {Hostel,HighMarks}:5✓ {CS,Hostel}:3✗
L₂ = {{HighStudy,HighMarks}, {Hostel,HighMarks}}
4
Generate Higher-Order Candidates & Repeat
Join L₂ pairs sharing (k-1) items to form C₃, then filter to L₃. Continue until no frequent itemsets remain.
C₃ = {HighStudy, Hostel, HighMarks} → check count → L₃ or ∅
Stop when Cₖ = ∅
5
Generate Association Rules from Frequent Itemsets
For each frequent itemset I with |I|≥2, generate all rules X→Y where X∪Y=I. Keep rules ≥ min_confidence.
From {HighStudy, HighMarks}:
HighStudy → HighMarks: conf = 7/8 = 87.5% ✓
HighMarks → HighStudy: conf = 7/7 = 100% ✓

Limitations & Alternatives
Apriori Limitations
  • Multiple database scans (slow on huge data)
  • Generates many candidate itemsets
  • Memory-intensive for dense data
  • Performance drops with low min_sup
Better Alternatives
  • FP-Growth: tree-based, 2 scans only
  • ECLAT: vertical data format, faster joins
  • AIS: earlier, simpler approach
  • Max-Miner: finds maximal itemsets
🌳 Concept
Multilevel Association Mining
Multilevel mining discovers association rules at various levels of a concept hierarchy. Rules at higher levels are more general but have stronger support; rules at lower levels are more specific but rarer — revealing granular patterns invisible at the top level.
Concept Hierarchy
Example Hierarchy — Education Dataset
Level 0 (All) → All Students
Level 1 (Dept) → CS, ECE, MECH, CIVIL
Level 2 (Gender) → Male, Female
Level 3 (Residence) → Hostel, Day Scholar
Level 4 (Performance) → High, Medium, Low

Level 1 rule: CS → HighPerformance (sup=45%)
Level 2 rule: CS ∧ Female → HighPerformance (sup=28%)
Level 3 rule: CS ∧ Female ∧ Hostel → HighPerformance (sup=14%)
Uniform vs Reduced Support
Uniform Min-Support
  • Same threshold at all levels
  • Simple to implement
  • May miss rare lower-level rules
  • Used in basic Apriori
Reduced Min-Support
  • Lower threshold at deeper levels
  • Captures more specific patterns
  • More computationally intensive
  • Better for real-world datasets
🔷 Concept
Multidimensional Association Mining
While single-dimensional rules involve one attribute (e.g. buys bread → buys butter), multidimensional rules involve two or more attributes simultaneously — revealing cross-attribute patterns that no single-column analysis can find.
Types of Multidimensional Rules
Inter-Dimension Rules
Each attribute appears once. e.g. age=young ∧ dept=CS → marks=high
Hybrid Rules
Some attributes appear multiple times at different levels. e.g. dept=CS ∧ gender=F → gpa=high ∧ award=yes
Heatmap Interpretation
Reading the Cross-Tabulation Matrix
Each cell (row=X, col=Y) = count of records where both X and Y are true.

Cell intensity ∝ record count — darker = more records

📐 Reference
Complete Metrics Reference
A comprehensive reference for all metrics used in association rule mining, with formulas, ranges, interpretation guides, and when to use each one.
All Metrics — Formulas & Ranges
Support(X→Y) = |X∪Y| / N Range: [0, 1] Higher = more frequent
Confidence(X→Y) = |X∪Y| / |X| Range: [0, 1] Higher = more reliable
Lift(X→Y) = Conf(X→Y)/Supp(Y) Range: [0, ∞) >1 = positive corr
Leverage(X→Y) = Supp(X∪Y)-Supp(X)·Supp(Y) Range:[-1,1] >0 = dependent
Conviction(X→Y) = (1-Supp(Y))/(1-Conf) Range: [0, ∞) ∞ = perfect rule
Jaccard(X,Y) = |X∪Y|/(|X|+|Y|-|X∪Y|) Range:[0,1] Similarity measure
Choosing the Right Threshold
High min_sup (>50%)
Very common patterns only. Few rules, high quality. Good for confirming expected patterns.
Medium min_sup (20-50%)
Balanced discovery. Most useful for educational/business datasets. Default recommendation.
Low min_sup (<20%)
Rare pattern discovery. Many rules, some noise. Use with high confidence to filter.
High conf (>80%)
Only very reliable rules. Use when prediction accuracy matters more than coverage.
